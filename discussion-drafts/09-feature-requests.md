---
title: "Feature Requests: how to write one that actually ships"
category: "Feature Requests"
---

Good feature requests include:

- the user story (who, what, why)
- the expected outcome
- what you tried already
- constraints (builder, hosting, workflow)
- screenshots or examples (redacted)

Bad feature requests:

- "make it better"
- "add AI magic"

If you can define success, we can build it.
